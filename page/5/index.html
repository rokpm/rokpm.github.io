谈谈项目开发流程

发表于 2013-11-09 | 分类于 产品经理

安卓APP已经开发完成了，下周进行最后一轮的测试，安排好后续的工作流程，推进上线计划，顺利的话，下下周就可以在各大应用平台推出了。

应该是8月23号接到老大从西藏打来的电话，让我负责这个小项目的所有事情，从原型到文档，到项目跟进管理，到最终的测试和上线。想不到一个小小的安卓手机APP居然花了7周左右的时间。但毕竟这是我第一个全权负责的项目，得总结一下，好为以后的项目开展做准备。

按流程一步步细说。

1、立项

按照一般的立项过程，首先得做用户需求调查和市场分析，看看用户在心理APP方面有那些需求，看看市场上是否有类似的产品，进行对比分析，最终确定这个产品是不是应该做，应该怎么做。很遗憾，在这个小项目中，这个步骤被直接省去了。没有任何调查，只有一个长途电话，然后我就立马开工。没有所谓的BRD、MRD文档，没有任何用户调查。可能这只是一个小项目，所以很多东西都简化的，但在以后的产品中得把这一点做好，做好用户调查，做好需求分析，让每个产品都有它存在的意义。

2、画原型

这恐怕是每个产品经理最开始做的事情吧，作为一个产品的设计者，原型图需要的不是多好看，多保真，需要的是把需要表达展示的内容展示出来，需要把基本交互做得更符合用户的操作习惯。很高兴的是我一开始就意识到了这个问题，没有花大把的时间来弄原型图的外观设计，那是设计师的事情，产品经理把原型画得太好，太精细，反而会影响UI设计师的灵感。

3、写PRD文档

PRD文档即产品需求文档。就是把原型图用文字语言描述给设计和技术，遗憾的是这个我做得非常不好。找借口的原因是我根本没有任何PRD文档，也没有人教我如何去写。我自己的本身问题也是最终要得问题，我没有意识到PRD文档的重要性。人们不喜欢去思考，所以你需要用清晰明了的语言去向他们解释这些功能是如何实现的，这就是PRD文档的作用，详细描述每个功能是如何实现的，然后技术照着做就行。详细、清晰、明了，把看PRD文档的人当傻瓜，抱着这种信念去写PRD文档才能得到大家的认同，才能减少沟通的时间成本。

4、设计

设计师绝对是这个世界上最苦B的职业，因为世界上每个人对于美的理解都不一样，我喜欢蓝色，你喜欢绿色，她喜欢紫色，所以每个人都能够在设计师的设计上指指点点，就像这次APP的设计，为了配合心理的文章内容，设计在配色上选了比较单调的颜色，结果因为老板喜欢红色，所以推翻重做改为红色。而作为一个不懂设计的项目负责人，我需要做的就是信任。给予设计足够的信任让她们自由发挥。第一个是因为我不懂，第二个则是我坚信每个人都有自己擅长的东西，设计擅长的就是设计出用户喜欢的色彩和布局，那为什么不去信任她们呢。

5、开发

产品经理得拿刀逼着开发做事。这个是我做这个项目的体验。没办法，开发资源紧张，所以你得跟他们定好进度，约定什么时候做完，提前去追问项目进度，在约定的时间之前一定得逼着开发把事情做完。之前的老大就是因为这个被老板开了。

6、细节

项目进展过程中总会发生你想想不到的问题。而这时候需要做的就是找到相关的负责人，跟其商量找到最合理，最迅速的解决办法。

 

先就到这里了，等APP测试完上线后可能会有新的体会，到时候再添加进去。

为什么要从事互联网

发表于 2013-08-23 | 分类于 产品经理

因为互联网改变了我的生活，而我也想参与其中去改变更多人的生活。

自从高中的时候看了一本《穷爸爸，富爸爸》，我便知道理财的重要性，看了很多相关的书籍，知道了很多理财的技巧，也养成了记账的习惯，但即使这样，除了做过一次一年的零存整取再也没做过其它的事情，为什么，因为麻烦，股票基金债券都需要你去银行或者证券公司开户，而且每个月只能结余一两百块钱的我也不好意思站在人家柜台前面。但阿里刚刚推出的余额宝却改变了现状，网上直接可以购买货币基金而且可以免各种手续费随时支取。于是我把所有的工资都放在了余额宝上，即使每天只有几毛钱的利息我却依然很开心，因为这只是开始，阿里的余额宝是一把钥匙，打开了这扇门。

Evernote让我不再担心没地方记笔记及笔记本太麻烦的问题。网络硬盘取代了我手上的U盘，手机里的相片上传到PC端随时可以查看，昨天360云盘赠送1TB的存储空间，果断领走。百度地图，对于路痴的我再也不用看那让人头大的纸质地图了。最近兴起的网络教育，多贝，网易云课堂，在家就可以聆听到大牛的讲座。

还有现在用的新浪云应用，每个月6块钱，我不用学习任何编程知识，不用劳心费力的去买流量、空间和域名，我就有了自己的个人博客，这可一直是我的梦想，因为互联网，所以才能轻松的实现。

还有太多太多可以举的例子。互联网简化了我的生活，让我可以更轻松的接触到自己想要的东西，也让我可以更关注自己现实的生活。

互联网改变了我的生活，而我也想参与其中，开发出新的产品去方便用户的生活。

逃离游戏公司

发表于 2013-07-20 | 分类于 产品经理

写对比分析的时候突然意识到一个问题。

每次和产品、策划、运营讨论问题，发现大家的思考起点都是从其它游戏出发，其它游戏如果，然后我们的游戏该如何。就比如我最近做的对比分析一样，通过分析其它成功游戏的内容进而来得出一些提升自己游戏的方法。

我承认做竞品分析没有错，能够轻松通过对比来发现自己的优势和劣势，进而对产品进行改进。但问题就在于我们每次在讨论游戏的时候都是在以这样的方式思考问题。QQ炫舞怎么怎么样，CF怎么怎么样,所以我们该怎么怎么样。于是一切的设计都是从别的游戏出发。可以借鉴但也不能完全照抄啊，或许游戏业内都是这样的吧，于是一款游戏火了，然后接二连三很多基本一样的游戏又都冒出来了。魔兽世界火了那么多年，中国的游戏厂商抄了那么多年。


或许就是中国的网游环境造成的吧，大部分用户都是低端用户，所以一款游戏不管怎么垃圾，只要迎合了大部分低端用户的口味它就能大赚，所以游戏越做越低端。在端游上，人傻钱多，于是大家都忘了创新吧~


或许这也是我为什么想逃离游戏公司的原因，公司的福利待遇足够好，但却满足不了自己深层次的渴望，我的渴望是自己能够参与设计一款能够方便普通用户生活的创造性产品，而这是游戏产品所不能给我的。

>

所有的服务都是对信息的处理

发表于 2013-07-08 | 分类于 产品经理

所有的服务都是对信息的处理。


手机主屏上有这几个应用：微信、挖财、Pocket、知乎、Evernote、ZAKER、Chrome、雅虎天气……


这几个应用是每天必定会打开的，占我使用手机90%的时间，仔细思考一下，发现这些APP拥有共同的特性：所有的APP都是对信息的处理。

通过微信来获取公众账号推送新闻是对信息的处理，通过挖财记账是对自己财务信息的处理，通过Pocket来保存未读的文章是对信息的处理，通过知乎、ZAKER、Chrome来获取新的知识是对信息的处理，通过Evernote来保存资料是对信息的处理……


那么其实可以说所有的服务都在完成一个使命：对信息进行处理。


对信息进行处理，那么最好的结果是用户使用最快捷的操作来获取最优质的信息。


那么进而可以推出，做出使用户满意的产品需要满足两个要素：一个是最快捷的操作，一个是最优质的信息。即让用户用尽可能短的时间获取尽可能优质的信息。


公交车上灵光一闪的念头，其实应该是互联网最基础的东西，只不过是通过自己思考得到了就感觉理解得更加透彻。

游戏产品六宗罪

发表于 2013-06-29 | 分类于 产品经理

随着时间的推移，接触到了项目深层次的东西，也开始慢慢发现了项目中存在的问题，以一个新手的身份从产品角度分析一下项目的失败之处。

1、忽视交互设计，导致重点缺失

作为一个客户端游戏，玩家每次进入游戏都需要点击大量的下一步，下一步，PPT一样枯燥的展示游戏中的内容，完全不考虑玩家感受。lobby界面单调的按钮展示，重点突出比赛而忽视其它休闲内容的建设，直接导致玩家只能不停的进行激烈的比赛，增加玩家的疲劳感，从而减少在客户端上停留的时间。

2、忽视玩家的实际需求

经常玩游戏逛BBS其实就可以很清楚明晰地知道玩家需要的扣篮大赛、三分球大赛、斗牛房。深层次的挖掘可以知道玩家需要的多样化的游戏模式，而不是简单重复的比赛，如果重视玩家的需求，那么项目的开发应该朝这个方向努力，但实际因为商业目的这些需求却被推后。

3、过分商业化，导致玩家反感

最新推出技能系统，而这个系统的推出完全是出于商业目的，据内部计算玩家升满级一个技能需要3000RMB，前期推出的有5个技能，后期会推出更多。由于2K原作优秀的计算方式，技能其实对游戏的平衡性影响并不大，但玩家却不这么认为，传统MMORPG游戏的收费模式是如果你不花钱去购买，那么你必须在游戏上花费大量的时间。但我们游戏中的系统却是你不花钱就无法进行技能升级。

4、缺失规范化的流程管理

作为QA的这一年中，项目经常会出现这样的问题，美术修改的相应的美术资源，但程序却不知道，直接导致游戏中美术资源缺失。PM反复强调某一功能的添加，而程序最后还是忘记把修改的代码添加到Gamelib中。这些直接导致版本错误而需要重新编译，每次编译都需要花费3~4个小时。规范化流程管理的缺失，导致项目不能按预期计划中进行。

5、将“添加功能”当做“产品提高”

产品基本成型后，为了对版本进行更新，提升玩家的新鲜感，策划会不停地向产品中添加新的功能，错误的认为“添加功能”就是“产品提高”，比如比赛数据的实时显示，花费大量人力物力添加的功能其实只满足的很小很小一部分用户的需求，不停的对产品做加法，最终导致产品臃肿不堪。

6、缺乏全局掌控

首推的就是inGame模块下各个弹窗，目前所有的弹出窗口基本上可以把整个屏幕布满，与2K原作比较，原作inGame下只有很小的NBA2K图标和一个小的scoreboard，整个屏幕上没有其它东西，以此让玩家能够专心沉浸在比赛当中，虽然online不一样，但也不至于弹出各种各样的窗口来干扰玩家的比赛。缺乏一个强势的PM从高层次规范产品，于是产品内容十分杂烂。

一年之内成为入门级的产品经理；

三年之内成为普通的产品经理，去帮助那些优秀的PM处理他们不愿意做的杂事；

五年之内成为优秀的产品经理，主导一个互联网项目，去完成20%最重要的事；

八年之内成为大神级的产品经理，做一款工具类应用，充分满足用户某方面的需求，小众但是精品（知乎、微信、Evernote、挖财、Pocket）。
