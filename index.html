我大脑中的回路-如何处理需求

发表于 2016-09-28   |   分类于 产品经理

做了快3年产品，关于如何做产品，已经开始慢慢构建起知识框架。按照《Learn How to Learn》中提到的概念，处理产品问题的方法已经在我大脑中产生了回路，当问题来临时，它会自己按照已产生的回路进行运转，最终得出解决方案。那么就写一篇文章，说明一下自己大脑中的回路。

知乎上有句名言，一切不问是不是就先问为什么的行为都是耍流氓。同样当一个需求来临时，不判断真伪，而直接提出解决方案，也是在耍流氓。工作中，这种耍流氓的行为很频繁。

我的解决方案是，多问为什么，找到核心问题。举个我实际遇到的例子，运营提出一个需求，我需要在给用户的短信里，加上客服的微信号。如果不加思考，很多人就直接去做了。可如果多问几个为什么，就能发现不一样的情况：

运营：”我们要在短信里加上客服的微信号！”

产品：“为什么要加微信号？”

运营：“因为需要与用户建立联系。”

产品：“为什么要建立联系。”

运营：“因为有些用户反应，不会用XX功能，加上微信，我就能告诉他怎么操作。”

只问2个为什么，就会发现，核心问题是用户不会用这个功能。一旦找到了核心问题，那么思路就会开阔很多，能想到很多解决方案。加微信是一个方法，但这个方法很低效，而且也很浪费人力成本，肯定不能采纳。

找核心问题，适合解决由用户、老板或同事提出来的小需求、小任务，多问为什么，从而达到去伪存真，找到真实需求的效果。而当提出一个大的需求时，需要当作一个项目来对待时，则需要用另外的思考方法，我常用的思考模型是借鉴《用户体验要素》中提出的互联网产品的5个层次。

书中将产品分为5个层次：战略层、范围层、结构层、框架层、表现层。表现层主要是设计师来负责，所以我常用前4个层次来思考、分解。

1.战略层

战略层上，需要同时思考两个问题：产品目标和用户需求。即我们要通过这个产品得到什么，我们的用户要通过这个产品得到什么。

作为一家商业公司，公司做任何事情，都有其目的。当老板吩咐要做一个项目时，首先就要思考，公司需要从这个项目中获得什么收益。这个收益是一个广泛的概念，可以是直接的利润，也可以是与利润相关的因素，例如降低人工成本，增加流程转化率、提高付费率等。

与此同时，在最开始的时候，我们就需要考虑用户需求，即开发这个项目能够帮助用户解决哪些问题、疼点。

理想情况下，产品目标和用户需求是完美融合的，但现实情况是两者偶尔会发生冲突，这时候就需要去考虑如何最大程度地消除两者带来的冲突。

2.范围层

进入范围层，需要考虑，为了实现产品目标和用户需求，我们需要实现那些功能，也就是所谓的做什么与不做什么。

我一般使用思维导图，将与之相关的所有功能点、需求点都罗列出来，依据我们上一层次中分析出来的产品目标和用户需求，将各个点划分优先级，所有优先级为P1的功能集合即是这个产品的MVP模型。依据MVP模型，确定后续的开发内容。产出这张思维导图后，我会与老板沟通，确定范围是否有遗漏，以及优先级划分是否合理。

3.结构层

前两个层次，我们解决了为什么要做以及做什么的问题。在结构层，我们开始处理具体的产品策划了。在移动端，大部分功能都是为了让用户实现某一个具体的目标，例如，支付模块就是为了让用户完成支付。所以在这一层次，我使用流程图的方式来说明用户从开始到实现目标的整个流程。

这样做的好处是能够清楚得看到用户完成目标所需要的步骤，进而可以思考如何优化这些步骤。并且在最终上线时，我们可以通过监测点监测到这些步骤间的转化率，从而发现问题并进行优化。

4.框架层

这一层次也就是原型阶段，产出的内容是原型文档。在我们的团队，为了优化产品经理的产出效率，产品团队不出PRD文档，那么原型成了唯一说明页面细节地方，所以我们的原型会有大量的标注来对各个位置进行解释。

虽然做产品的最初阶段，我就在画原型，但因为不是设计的知识积累不够，所以并不太擅长这部分。我采用的原则就2点，强调重点信息，简化所有可以简化的信息。

以上就是我在处理需求时，所采用的思考模型。由于视野、经历有限，还存在大量“我不知道我不知道”的内容，再接下来的时间里，我也会在工作中继续完善我的产品知识框架。

谈谈多肉

发表于 2016-09-25   |   分类于 生活感悟

内心一直对自己的居住环境要求颇高，而养些活物，无论是动物或植物，都能为房间增加一些生机，所以我经常会养些东西。

养过蜗牛，忘了关紧瓶盖，结果找不到了，半年后搬家，发现它在桌子底下，不过已经风干了。养过小狗，白天去上班把它关在小房间里，没几天，精神萎靡不振，有点抑郁症的倾向，于是送人了。养过仙人球，似乎还挺不错，养了好几个月。刮风的时候把花盆吹到地上，碎了一地，这时才发现，那仙人球的根茎已烂得不成样子，原来它早死了，只是死得不太明显。

前几个月，开始养多肉，淘宝10块钱包邮弄了3小株，每天悉心照顾，半个月不到，卒。朋友又给了1盆，1个月不到，现在也快不行，奄奄一息。

完全无法理解为什么仙人球、多肉这么好养的植物，也会死在我手里。于是找到张大妈之前推荐的一篇讲多肉的帖子。看完之后，才发现事实跟我以为的完全不一样。

多肉居然是生活在干旱地区的植物！

这是一件让我很是震惊的事实。要知道多肉长得这么可爱、粉嫩，常常与青苔一起出场，让我以为它长在茂密、湿润的森林里与蘑菇为伴。可曾想，它生长在干旱地区的石头缝里，枝叶肥厚不过是为了储存更多的水分。

就像身边的小女生，长得小巧柔弱，让你顿生怜惜，结果却是力能扛鼎的金刚芭比。

我把它移居到阳台上，让它充分接触大自然，享受日晒、风吹和雨淋，等到水分干透时，再人工给它浇水。有时候太过用心也是一种错误，不知道这次会不会有不一样的结果。

读书笔记-《如何阅读一本书》

发表于 2016-07-21   |   分类于 读书旅行

按照我的理解，作者将这本书分成3个部分：第一部分谈成为一个高阶的阅读所需要的思想认识，更主动，使用更多的技巧，提出问题与作者互动，以及阅读所追求的目标；第二个部分谈阅读的四个层次，基础阅读、检视阅读、分析阅读和主题阅读；第三部分谈不同类型的书，例如实用型、想象文学等书的阅读方法。

我是如何招产品助理的？

发表于 2016-04-12   |   分类于 产品经理

作为一个仅2.5岁的产品经理，来谈谈我是如何招产品助理的。


为什么需要一个产品助理？

随着公司业务发展，产品新增了很多分支，导致我没有时间和精力处理每个细节，于是需要一个产品助理来帮助我分担部分工作：测试（PS：创业公司没有专职测试），补充原型和文档说明，与设计、产品沟通推进项目进展，独立完成部分功能的策划。


大部分应聘产品助理求职者都没有太多产品相关经验，他们之中要么刚毕业，要么转行过来的。而我的职责就是从这群毫无产品工作经验的人中挑选出能够胜任公司产品工作的人。毕竟，大多数公司，产品经理主导着产品的开发设计，虽然是产品助理，但也对项目起着很关键的作用。



一、看是否热爱做产品？

为了成为一名产品经理，你付出了什么努力？

看了哪些产品相关书籍，里面讲了哪些内容？写了多少分析文章？做了哪些些练手的小项目（Axure画个QQ登录界面也算）？上了哪些线上/线下课程？

什么？这些你都没做？好吧，你肯定不热爱产品，要不你怎么什么付出都没有。只是装了满手机的APP，不算热爱产品好吗？




二、看是否有能力做产品？


1.看简历

我们平常用的东西都是产品，简历当然也是，对于简历这个产品而言，面试官就是用户，我的需求在招聘上都写的清清楚楚，你的产品能满足我的需求吗？重点信息是否突出，非重点信息是否删除（普通话等级证书这些），排版格式是否清楚明了等等。所以，使用那些通用的简历模板基本是不会有面试的机会的。


2.看沟通能力

能否听明白我的话，能否让我明白你的话。作为产品经理，需要迅速捕捉到重点、表达出重点。销售的沟通能力是为了让信息更具吸引力，而产品的沟通能力更多是为了减少信息传递过程中的失真，所以你性格是否内向外向完全没关系。


3.看是否具有产品思维

我面试的时候喜欢问面试者这样一个问题：如何给盲人设计一辆自行车。这道题其实就是考察面试者是否具有基本的产品思维。看你是从什么角度去考虑这个问题。不过很多面试者听完之后就直接跟我说自动驾驶、路面铺设专用轨道等等。


三、是否主动追求个人成长？

职业规划是什么？打算怎么样成长起来？我需要你能快速成长起来，帮我分担更多的事情。但身处创业公司，我没办法像大公司那样呵护你成长，我只能让你一边做事情一边自己摸索，必要的时候可以请教我。我可以把我知道的所有东西都教给你，前提是你得学会主动。


以上就是我招产品助理的判断标准。

一.产品经理的工作职责（产品经理在做什么？）

1.市场调研（评估产品机会，MRD）

需求分析、用户研究、熟悉领域

2.产品定义及设计（定义要开发的产品，PRD）

产品设计、文档撰写

3.项目管理

沟通协作、项目管理、熟悉技术、团队管理、新人入门

4.产品宣传

5.产品市场推广

产品运营、数据分析

6.产品生命周期管理

产品规划、业务规划

二.产品经理的岗位要求（产品经理需要什么样的技能/素养？）

1.沟通能力

2.无授权领导能力（正直、信心、责任感）

3.学习能力

4.商业敏感度（商业技能）

5.热爱产品（对产品的热情）

6.注重细节，追求完美（用户立场）

7.日常产品管理能力（注意力）

8.智力：解决问题的能力

9.职业操守

10.运用技术的能力

11.时间管理

三.如何成为产品经理

1.先在本职工作上找到与产品有关的事情做一些尝试，并且考虑先从产品经理周边的职位（需求分析师、运营专员、项目经理）做起。

2.培养产品感觉。


 

【课前预习】90天[产品家]关联阅读实践训练营

发表于 2016-04-10   |   分类于 产品经理

Q1.你是怎么入行的，为什么要当产品经理？

其实一直以来我的梦想是成为一名程序员，用自己的代码改变世界的。实习时抱着这个梦想来到魔都进入一家NB的游戏公司做测试，想慢慢往技术的方向转。但在一年的实习经历中，发现大部分程序员并没有决策权，只是按照已经制定好的方案去实现别人的想法，而我又是个想法特别多的人。碰巧在知乎逛的时候了解到了产品经理这个角色，选定好职业方向后，就看知乎上的文章，把能找到的好书都看了一遍，写写总结，练习工具的使用。最终毕业的时候，成功跳槽，来到一家创业公司担任产品助理。

Q2.你心中优秀的产品经理具有哪些素质？

沟通能力：能正确的理解别人的意思，能正确的表达自己的意思，并能快速推动项目的进行。

学习能力：能够发现自己的不足，并通过学习来弥补自己的不足。

发现问题及解决问题的能力：能找到正确的问题，能提出正确的解决方案。

当然最重要的应该是商业模型的搭建及完善，很多时候，产品的成功与否从产品开始之前就已经决定了

Q3.你欣赏并向往成为的产品经理是谁？

纯银。最近一段时间看了他的很多文章，虽然对他提到的产品模型理论不怎么理解，但还是很崇拜和向往这种工作状态。

Q4.这些年当产品经理的过程中，能力上有哪些成长，哪些欠缺，有什么好的成长秘籍？

遇到问题，能快速的定位问题，并提出正确的解决方案，然后推动方案的实现。这个应该是我成长最大的地方。

而最大的欠缺应该是对用户需求的把握，虽然这应该是产品经理最重要的能力，但总感觉自己在离用户很远的地方，只能模糊的感受到他们的存在，却不能深入的了解和理解他们。这些也是我给自己打70分的原因，已经及格，但离优秀还有一段距离。

Q5.审视一下自己，作为产品经理，哪些地方希望补全和加强？

搭建完整、系统的产品框架。这也是我参加这次的学习最主要的目的。这几年一致都在不停地做产品，很少停下来去思考、总结、规划自己的工作。这次正好跟着大家一起把这几年的工作经验系统的整理下。知道自己的不足，以及未来需要掌握的内容，一点点慢慢积累。

读书的我

发表于 2015-09-13   |   分类于 个人提升

看了看记录，今年到现在读了32本书，虽然与战隼这种大神相比，我还差太远，但从身边来看。大部分人都会把我归类成爱读书的那类人。

大部分中国人不爱读书是情有可原的，应试教育太过于功利，看课外书还不如做套卷子，资源太过匮乏，很少有好书看，整个教育阶段也没有任何引领过我们如何读书。于是大部分人毕业后就不再看书。

为什么我会经常看书？

大学之前，读书是为了自我麻痹。无论我们是否有状态学习，老师都会把我们禁锢在教室里面，不想做物理题，那就看看课外书呗，至少也是在学知识不是吗？虽然这些知识学校不考。

大学时读书是为了打发时间，老师课堂上讲的东西总也听不懂，索性不听了，看从图书馆里借来的书。

刚出来工作时，读书是为了提升专业技能，选了产品经理这个职位，但却没有导师来带，于是遇到问题就从书中寻找答案，是这些书教会了我如何做产品，虽然书和现实工作还是有很大差异。

现在看书，开始往开启心智的方向努力。我接触的世界太小，我认识的牛人也太少。渐渐的形成了自己独特的思维方式，也越来越发现每个人的思维方式都是不一样的，同样的一件事情在不同人的眼里居然能产生不同的价值，同样一个问题，不同的人居然有不同的解答方式，并没有所谓的标准答案。而读书能让我窥探一下作者的思维方式、心智模式。

除了自我麻痹，打发时间、专业技能提升、心智开启，这三种读书目的是并存的。最终我追求的是多方面的个人成长。
